Does it makes sense resetting CSS in 2025?

A look at the past, present, and future of browser styling

Why CSS Resets were born?

  • Early browsers had inconsistent default styles
  • Margins, paddings, font sizes varied wildly
  • Resets ensured a clean and predictable baseline

What's the philosophy of CSS Resets?

  • Erase all default styles
  • Manually reapply only what you want
  • Gain control over layout and spacing
  • Ensure consistent rendering across browsers

What are the main concerns with CSS Resets?

Hidden costs of resetting already working stuff

#0 - Accessibility: CSS Resets can be harmful to users who rely on keyboards for navigation

#1 - Redundancy: CSS Resets and Every element that is “zeroed” must then be redefined

#2 - Debt: CSS Resets are technical and design debt at the same time

#3 - Loss: CSS Resets make loss of useful defaults (e.g. system colors, focus styles, button behavior)

To reset or not to reset?

Takeaway

CSS resets were essential in the past, today we have better tools... and better defaults
